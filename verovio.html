Compare 3 elements in the MusicXML file to the corresponding ones in MEI:

  • MEI has more meta data can encode not only information such as title and composer it additionally can take information such as arranger contributor editor and lyricists, this allows for a more detailed encoding of a piece
  • Additionally it allows for a link to be made between the MEI and bibliographic data with library databases and bibliographic databases
  • Mei is a more condensed version - with many abbreviations while musicXML does not, this makes musicXML more reader friendly at first glance but a lot longer, while MEI is more condensed and uses specialist terms to MEI in order to read and interpret the information.
  • A third difference between the two is that musicXML is more widely recoginised within music notation software so it is more accesible and has the ability to be readd and interpreted by and across different notation softwares
